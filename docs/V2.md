# Regulatory Assistant — V2 (Android)

Цей застосунок **оновлено до V2**: функціонал з пакетів feedback / `CURSOR_PROMPT` інтегровано в **`com.example.myapplication2`**, без паралельного пакета `com.regulation.assistant.*` (щоб не дублювати ViewModel, Room і каталог ніш).

## Версія збірки

- У **Gradle**: `versionName` / `versionCode` у `app/build.gradle.kts`.
- У **UI**: Налаштування → **About** показує рядок на кшталт `Version 2.2 · V2 (unified engine)`.

## Що входить у V2

| Область | Опис |
|--------|------|
| **Пошук (research)** | Глибокі інструкції для AI по нішах: `NichePromptEnrichmentCatalog` → блок у `RemoteSearchRepository.buildMainPrompt`. |
| **Постійні правила** | `StandingRequirementsCatalog` (у т.ч. DPIA, NIS2 management, санкції, HACCP, CSRD double materiality). |
| **Календар** | Батчі ніш, ліміт подій на відповідь, стійкий парсинг JSON, retry/timeout для API — `RemoteCalendarRepository`, `GrokApi`. |
| **Конфіг** | `assets/regulation_config.json` + `RegulationConfigLoader`. |
| **Нотатки** | Room (`EventNotesRepository`), не SharedPreferences. |
| **Глосарій** | `RegulatoryGlossary.kt` (крос-секторні терміни); у UI також є **Regulatory Glossary** у Tools / Profile / Dashboard. |
| **Експорт** | Share + «Додати в календар» на картці події — `RegulatoryEventExport`. |
| **Сповіщення** | `RegulatoryNotificationService` + receiver (після успішної генерації календаря). |

Деталі імпорту з десктоп-пакетів: `docs/imports/FEEDBACK_V2.md`, `docs/imports/FILES2_PACKAGE.md`.

## Що свідомо не копіювали з zip

- Окреме дерево **`com.regulation.assistant`** і заміна всього `CalendarViewModel` на пакетний варіант.
- **`UserNoteManager` (Gson + SharedPreferences)** — замість нього Room.

Якщо потрібна **буквальна** структура з `cursor-package.zip`, це окремий рефакторинг з мерджем конфліктів; поточний репозиторій уже відповідає **функціональному V2**.
